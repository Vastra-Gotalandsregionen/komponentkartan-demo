<vgr-icon> används för att visuellt förstärka en handling eller ett meddelande.

Komponenten icon <vgr-icon> använder sig av ikonerna som kommer från Font Awesome (free). Komponenten är en anpassad variant av Font Awesomes Angular-komponent och stöder dens Basic funktionalitet angular-fontawesome features (basic). När <vgr-icon> används ska dock implementationen följa API nedan.

De ikoner som går att använda hittar du här: https://fontawesome.com/icons?d=gallery&m=free.




Design

Ikoner bör användas i kombination med text i första hand, då ikoner kan vara svåra att tolka när de står själva. De bör alltså användas för att förstärka ett budskap snarare än att förmedla det.

I vissa fall är det lämpligt att använda ikonen självstående, t.ex. för att förmedla en status såsom “granskad” eller “godkänd”. Det är rekommenderat att använda tooltip för att förtydliga vad ikonen betyder.

Ikoner som förmedlar information behöver ha en hög kontrast mot sin bakgrund för att kunna läsas av användare som tittar på skärmen. Därför är det endast tillåtet att använda de primära färgerna i färgkartan på ikonerna. Undantaget är när ikonen ska reflektera ett inaktivt läge, då är det tillåtet att ha en lägre kontrast. Färgerna som är tillåtna begränsas av komponenten.

Namn Beskrivning Exempel
icon: string Ikonens textkod. Möjliga värden är koder enligt Font Awsomes dokumetation för gratisikoner. Default inget värde. [icon]="'star'"
solid: boolean Sätter om ikonen skall vara solid. Möjliga värden är true (solid) eller false (regular). Default är true. [solid]="false"
size: string Ikonens storlek. Möjliga värden är 'xs' (.75em), 'sm' (.875em), 'lg' (1.33em), '[n]x' ([n]em) etc där [n] är ett tal ex '2x' (2em). Default tom sträng (följer fontstorleken). [size]="'xs'"
color: string Sätter ikonens färg. Möjliga värden är 'primary' (följer temat), 'text' (följer textfärgen), 'success', 'error'. Defaultvärde är 'primary'. [color]="'text'"
disabled: boolean Visar ikon i som inaktiv. Default är false. [disabled]="true"
ariaLabel: string Möjlighet att sätta en eget röstmeddelande på ikonens area-label. Default tom sträng . [ariaLabel]="'min favorit'"
fixedWidth: boolean Låser storleken till ett fast värde. Default är false. [fixedWidth]="true"
rotate: number Roterar ikonen i antal grader. Tillåtna värden är 90, 180 och 270. Default är 0. [rotate]="90"
flip: string Spegelvänder ikonen. Tillåtna värden är 'horizontal', 'vertical' och 'both'. Default är tom sträng. [flip]="'vertical'"
spin: bolean Gör att ikonen spinner. Tillåtna värden är true eller false. Default är false. [spin]="true"
pulse: bolean Gör att ikonen spinner på ett snabbare och pulserande sätt. Tillåtna värden är true eller false. Default är false. [pulse]="true"
border: bolean Lägger på en tunn border runt ikonen. Tillåtna värden är true eller false. Default är false. [border]="true"
pull: string Skuter ikonen åt vänster eller höger. Tillåtna värden är 'left' och 'right'. Default är tom sträng. [pull]="'right'"

När ikonen används för att förstärka en text behöver skärmläsaren inte läsa upp ikonen, eftersom den är redundant. I de fall då ikonen används själv behöver aria-label sättas för att skärmläsaren ska kunna läsa upp den.

Enligt WCAG 2.1 ska icke-textelement ha en hög kontrast mot sin bakgrund så att personer med nedsatt syn kan läsa och tolka dem. För att uppnå detta kriterie ska ikonerna endast ha färger med god kontrast mot bakgrunden. I Komponentkartans färgkarta är dessa färger de primära färgerna, success, fail och textfärgen.

Mer information om kontrast finns på: https://www.w3.org/WAI/WCAG21/Understanding/non-text-contrast.html
