Notifiering som läses direkt upp av en skärmläsare


Tryck på knapparna nedan för att få fram en notifiering med alert som läses upp av eventuell screenreader

Ett felmeddelande! Ta fram notifiering Rensa notifiering
Namn Beskrivning Exempel
standalone : boolean Skall notifieringen använndas utanför vgr-grid [standalone]="'true'"
type : string Styr vilken notifiering man vill ha (ikon och färg som automatiskt läggs till). Tillåtna värden är 'success', 'error', 'warning', 'info' och 'comment'. [type]="'error'"
width : string Sätter bredden på notifieringen [width]="'300px'"
height : string Sätter höjden på notifieringen. Det som sätts är min-height då den behöver kunna växa åt något håll ifall innehåll är för stort [height]="'100px'"
noICon : boolean Om man ej vill ha ikonen som annars följer med automatiskt när man anger type [noICon]="'true'"
isAlertNotice : boolean Sätter man om man vill att notifieringen skall läsas upp direkt av en screenreader. Denna skall användas med varsamhet då endast meddelanden av hög prioritet bör läsas upp och gå före ordningen av uppläsning på en sida. Läs gärna denna artikel om du är osäker på om den skall användas https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/alert_role. Obs!När man använder ngIf för att ta bort ett meddelande med isAlertNotice så behöver man först ta bort isAlertNotice och sedan i en setTimeout ta bort notifieringen. Annars läses notifieringen upp både vid visande av notifieringen och vid borttagning av notifiering. Default är false. [isAlertNotice]="'true'"
Genom att sätta isAlertNotice till true så triggar man att notifiering kommer läsas upp direkt av en screenreader utan att användaren behöver ha fokus på notifieringen
