Dokumentation för monthpicker

Översikt

<vgr-monthpicker>


Ger möjlighet att välja år och månad. Ställs in med min/maxdatum som kan vara hela eller del av år.


Förändras med tema: Ja (Knapp, header samt markering av månad)

Exempel

Default datumformat

Med defaultinställningar sätts min- och maxdatum till innevarande år.


MMMM

Här är selectedDate från 2015. minDate blir då default 2015, maxDate blir innevarande år.
DatumFormat: MMMM


Med min- och maxdatum, obligatoriskt

Här är minDate satt till 2015-05-01 och maxDate är 2016-03-31. Från 2015 och fram till nuvarande år så visas samtliga månader, men endast månaderna mellan maj 2015 - mars 2016 är valbara. . Fältet är required och därför ges valideringsfel om inget värde väljs.


Disabled


Readonly

Här är kalendern satt i readonly-läge och selectedDate satt till 2015-01-13.

Datumformat MMMM


Senast valt datum: {{lastSelectedDate | date : 'yyyy MMM'}}

API

Namn Beskrivning Exempel
selectedDateFormat : string defaultvärde: 'MMM yyyy', här kan man sätta hur datumet skall presenteras i dropdownen [selectedDateFormat]="'MM yyyy'"
toolTipDateFormat : string defaultvärde: 'MMMM yyyy', här kan man sätta hur datumet skall presenteras när man hovrar över det [toolTipDateFormat]="'MMMM'"
minDate : Date Defaultvärdet är innevarande år. Om man förvalt ett datum innan dagens år så sätts minDate till det förvalda året. Om man satt ett förvalt datum som är efter dagens år så sätts dagens år till minDate [minDate]="minDate"
maxDate : Date Defaultvärdet är innevarande år. Om man förvalt ett datum innan dagens år så sätts maxDate till det innevarande året. Om man satt ett förvalt datum som är efter dagens år så sätts det förvalda året till maxDate. [maxDate]="maxDate"
selectedDate : Date Property för att kunna skicka in ett förvalt datum till komponenten [selectedDate]="forvaltDatum"
disabled : boolean true/false, defaultvärde: false. Möjlighet att sätta disabled på månadskalendern [disabled]=true
required : boolean Anger om värde är obligatoriskt eller ej. Om required=true och inget värde valts när komponenten tappar fokus ges ett validerings fel. Vid val av värde uppdateras validering. Vid fokus indikeras att valideringsfel redigeras. För att trigga validering utifrån använd validate(). [required]="true"
validate() : IValidationResult Anropas utifrån för att validera komponenten. ValidationResult.isValid indikerar om innehåll är giltigt eller ej. ValidationResult.errorMessage ger eventuellt fel. const result = component.validate()
selectedDateChanged : EventEmitter<Date> Event som triggas när man väljer ett nytt datum (selectedDateChanged)=onselectedDateChanged($event)
readonly: boolean Om monthpicker är satt som readonly så ska bara textfält med datum visas. Om datum är förvalt så visas det som är satt i selectedDate annars visas ingenting. Datumet är formaterat på selectedDateFormat. [readonly]="true/false"
